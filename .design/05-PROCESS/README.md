## 🎯 How to Use

**Starting a Sprint?**
1. Read `SPRINT_PLANNING.md`
2. Use `TEMPLATES/PLAN_TEMPLATE.md`
3. Follow `SPRINT_LIFECYCLE.md`

**During Sprint?**
1. Follow `TESTING_STRATEGY.md` (TDD)
2. Check `CODE_COVERAGE.md` regularly
3. Review `ARCHITECTURE_REVIEW.md` before major changes

**Ending Sprint?**
1. Verify `DEFINITION_OF_DONE.md`
2. Fill `TEMPLATES/FINAL_REPORT_TEMPLATE.md`
3. Complete `RETROSPECTIVE_TEMPLATE.md`
