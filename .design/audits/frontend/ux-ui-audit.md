# UX/UI Audit

**Scope:** User experience, interface design, usability, design system

## Design Consistency

- [ ] Design system documented
- [ ] UI components consistent across application
- [ ] Color palette defined and followed
- [ ] Typography scale defined and consistent
- [ ] Spacing system consistent (8px grid, etc.)
- [ ] Icon set consistent
- [ ] Button styles consistent
- [ ] Form element styles consistent
- [ ] Design tokens used
- [ ] Brand guidelines followed

## Visual Design

- [ ] Visual hierarchy clear
- [ ] Whitespace used effectively
- [ ] Alignment consistent
- [ ] Contrast sufficient for readability
- [ ] Color usage purposeful
- [ ] Visual balance maintained
- [ ] Design feels modern and professional
- [ ] Dark mode implemented (if applicable)
- [ ] Responsive design polished
- [ ] Print styles considered (if applicable)

## Navigation

- [ ] Navigation structure intuitive
- [ ] Primary navigation easily accessible
- [ ] Breadcrumbs for deep navigation
- [ ] Search functionality available (if needed)
- [ ] Back button behavior correct
- [ ] Navigation state persists appropriately
- [ ] Mobile navigation usable
- [ ] Navigation labels clear
- [ ] Current location indicated
- [ ] Navigation shortcuts available (keyboard)

## Layout & Responsive Design

- [ ] Layout adapts to screen sizes
- [ ] Mobile-first approach used
- [ ] Breakpoints appropriate
- [ ] Content readable on all devices
- [ ] No horizontal scrolling (unless intentional)
- [ ] Touch targets sized appropriately
- [ ] Landscape orientation considered
- [ ] Tablet experience optimized
- [ ] Desktop experience optimized
- [ ] Ultra-wide screens considered

## Forms & Input

- [ ] Form layout logical and clear
- [ ] Labels clearly associated with inputs
- [ ] Placeholder text helpful (not replacing labels)
- [ ] Input validation clear and immediate
- [ ] Error messages specific and helpful
- [ ] Success feedback provided
- [ ] Required fields clearly marked
- [ ] Help text available where needed
- [ ] Autocomplete enabled where appropriate
- [ ] Multi-step forms show progress

## Buttons & Actions

- [ ] Primary actions clearly distinguished
- [ ] Secondary actions de-emphasized
- [ ] Destructive actions require confirmation
- [ ] Button labels action-oriented ("Save Changes" not "Submit")
- [ ] Loading states shown for async actions
- [ ] Disabled state visually clear
- [ ] Button sizes appropriate for importance
- [ ] Icon buttons have tooltips
- [ ] Hover states provide feedback
- [ ] Active/pressed states visible

## Feedback & Communication

- [ ] Loading indicators for async operations
- [ ] Progress indicators for long operations
- [ ] Success messages clear and timely
- [ ] Error messages helpful and actionable
- [ ] Empty states designed and helpful
- [ ] 404 pages helpful
- [ ] System status visible when relevant
- [ ] Confirmation dialogs for destructive actions
- [ ] Tooltips provide helpful information
- [ ] Notifications not intrusive

## Content & Copy

- [ ] Microcopy clear and concise
- [ ] Tone of voice consistent
- [ ] Technical jargon minimized
- [ ] Help text available where needed
- [ ] Error messages in plain language
- [ ] Content scannable (headings, bullets, short paragraphs)
- [ ] Call-to-actions clear
- [ ] Placeholder text helpful
- [ ] Empty states have actionable copy

## Information Architecture

- [ ] Content organized logically
- [ ] Grouping and categorization clear
- [ ] Information hierarchy clear
- [ ] Related content grouped
- [ ] Progressive disclosure used appropriately
- [ ] Cognitive load minimized
- [ ] User mental models considered
- [ ] Findability optimized

## Interaction Design

- [ ] Interactions intuitive
- [ ] Gestures consistent with platform conventions
- [ ] Drag-and-drop works smoothly (if used)
- [ ] Swipe actions clear (mobile)
- [ ] Hover effects provide feedback
- [ ] Click/tap targets sized appropriately
- [ ] Double-click/tap prevented where inappropriate
- [ ] Long-press actions clear (mobile)
- [ ] Keyboard shortcuts documented

## Performance Perception

- [ ] Perceived performance optimized
- [ ] Skeleton screens for loading
- [ ] Optimistic UI updates where appropriate
- [ ] Animations smooth (60fps)
- [ ] No layout shifts during loading
- [ ] Progressive enhancement used
- [ ] Graceful degradation for slow connections
- [ ] Offline experience considered

## Data Display

- [ ] Tables readable and sortable
- [ ] Lists formatted consistently
- [ ] Cards use consistent layout
- [ ] Data visualization clear and accurate
- [ ] Charts/graphs labeled clearly
- [ ] Large datasets paginated
- [ ] Filters intuitive and accessible
- [ ] Search results relevant and formatted well
- [ ] Date/time formats consistent
- [ ] Number formats consistent (currency, percentages, etc.)

## Modals & Overlays

- [ ] Modals used appropriately (not overused)
- [ ] Modal purpose clear
- [ ] Modal size appropriate to content
- [ ] Easy to close (X button, ESC, click outside)
- [ ] Background dimmed/blurred
- [ ] Focus trapped in modal
- [ ] Scrolling behavior correct
- [ ] Nested modals avoided (or handled well)
- [ ] Mobile modals full-screen if needed

## Notifications & Alerts

- [ ] Notification type appropriate (toast, banner, modal)
- [ ] Notifications dismissible
- [ ] Notifications don't block critical content
- [ ] Notification duration appropriate
- [ ] Multiple notifications handled gracefully
- [ ] Notification position consistent
- [ ] Notification priority clear
- [ ] Notification actions clear

## Images & Media

- [ ] Images sized appropriately
- [ ] Image quality good (not pixelated)
- [ ] Alt text provided
- [ ] Lazy loading implemented
- [ ] Placeholder images while loading
- [ ] Image aspect ratios maintained
- [ ] Responsive images used (srcset)
- [ ] Video controls accessible
- [ ] Media doesn't autoplay with sound

## Typography

- [ ] Font choices readable
- [ ] Font sizes appropriate
- [ ] Line height comfortable (1.5 for body text)
- [ ] Line length reasonable (50-75 characters)
- [ ] Heading hierarchy clear

- [ ] Font weights used purposefully
- [ ] Text color sufficient contrast
- [ ] Web fonts loaded efficiently

- [ ] Animations purposeful (not gratuitous)
- [ ] Transitions smooth
- [ ] Animation timing feels natural
- [ ] Reduced motion preference respected
- [ ] Animations don't cause layout shifts
- [ ] Loading animations not distracting
- [ ] Page transitions smooth (SPA)
- [ ] Micro-interactions polished
