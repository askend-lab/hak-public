## 🎯 Key Learnings

1. **jest-cucumber requires all scenarios** - Can't just skip, need stub step definitions
2. **ts-jest needs npm install first** - Transform fails without the package
3. **import.meta.env is Vite-only** - Jest can't parse it, need separate config or Vitest
4. **Zustand is simpler than Redux** - Clean store pattern with `create<State & Actions>()`
5. **Zod schemas export types** - `z.infer<typeof Schema>` gives TypeScript type
6. **Phase-based PRs work well** - One PR per phase keeps changes focused
7. **Golden use case is effective** - Full vertical slice demonstrates all patterns

## 🔧 Technical Details

### Zod Schema Pattern
```typescript
export const TaskSchema = z.object({
  id: z.string(),
  name: z.string().min(1),
  entries: z.array(TaskEntrySchema),
});
export type Task = z.infer<typeof TaskSchema>;
```

### Zustand Store Pattern
```typescript
export const useTasksStore = create<TasksState & TasksActions>((set) => ({
  tasks: [],
  setTasks: (tasks) => set({ tasks }),
  addTask: (task) => set((state) => ({ tasks: [...state.tasks, task] })),
}));
```

### Auth Context Pattern
```typescript
export function useAuth(): AuthContextValue {
  const context = useContext(AuthContext);
  if (!context) throw new Error('useAuth must be used within AuthProvider');
  return context;
}
```

### SingleTableLambda Key Design
```typescript
// PK: USER#{userId}, SK: TASK#{taskId}
await apiRequest('/save', {
  pk: `USER#${userId}`,
  sk: `TASK#${taskId}`,
  data: task,
  type: 'private',
});
```
