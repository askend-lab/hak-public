## 🚨 Question 1: Pronunciation Variants

### The Question
**How do we obtain different pronunciation variants for each word?**

### Why Critical
The prototype shows multiple pronunciation options for each word (e.g., ku'idas, kui'das, k'uidas). This is a core feature, but we don't know where these come from.

### Possible Scenarios

**A) Stress marker component provides variants**
```
Input: "kuidas"
Output: [
  { variant: "ku'idas", stress_pos: 1, context: "default" },
  { variant: "kui'das", stress_pos: 2, context: "emphasis" },
  { variant: "k'uidas", stress_pos: 0, context: "dialectal" }
]
```
- **If YES:** Estimate accurate, straightforward implementation
- **Time impact:** 0 hours additional
- **We need:** API documentation

**B) Variants must be generated by linguistic rules**
```
We implement rules:
- Estonian stress rules
- Morphological analysis
- Context detection
```
- **If YES:** Significant additional complexity
- **Time impact:** +40-60 hours
- **Cost impact:** +€2,800-4,200
- **We need:** Linguistic expertise or rules documentation

**C) Variants from pre-built database**
```
Database lookup:
SELECT * FROM word_variants WHERE word = 'kuidas'
```
- **If YES:** Need database, maintenance overhead
- **Time impact:** +20 hours (setup)
- **We need:** Database dump or API

**D) They don't exist - manual creation only**
```
User must manually create all variants
No automatic suggestions
```
- **If YES:** Major UX downgrade from prototype
- **Time impact:** -30 hours (simpler)
- **Impact:** Product is less useful

### What We Need
- [ ] Access to stress marker component to test

- [ ] Example inputs/outputs

## 🚨 Question 3: Word-Level Audio Playback

### The Question
**When user clicks a word to hear it, how do we play just that word?**

### Context
Prototype shows:
1. User clicks word "kuidas"
2. Multiple pronunciation variants appear
3. Each has a 🔊 button
4. Clicking plays just that word

### Possible Approaches

**A) Extract from full sentence audio**
```
If we have word timing:
extract_segment(full_audio, start=0.5s, end=1.2s)
```
- **Pros:** No extra synthesis
- **Cons:** Includes context (coarticulation effects)
- **Feasible?** Only if we have timing (Question 2)

**B) Synthesize words on-demand**
```
User clicks "kuidas" variant 2:
→ synthesize("kui'das")
→ return audio
```
- **Pros:** Clean, isolated word
- **Cons:** Delay (queue), many synthesis calls
- **Storage:** Cached after first request

**C) Pre-synthesize all words**
```
When sentence submitted:
1. Synthesize full sentence
2. For each word:
   - Synthesize default variant
3. For each variant (if known):
   - Synthesize that too
```
- **Pros:** Instant playback
- **Cons:** Massive synthesis load, storage
- **Example:** 
  - Sentence with 5 words
  - Each word has 3 variants
  - Total: 1 + (5 × 3) = 16 audio files per sentence

### What We Need
- [ ] How many variants per word typically?
- [ ] Client preference: speed vs storage?
- [ ] Merlin queue capacity?
- [ ] Decision: A, B, or C?

## 🚨 Question 5: "Use in Sentence" Behavior

### The Question
**When user selects a word variant and clicks "Use in sentence", what exactly happens?**

### From Prototype
1. User clicks word "kuidas" in sentence "Tere, kuidas läheb?"
2. Sees variants: ku'idas, kui'das, k'uidas
3. Selects "kui'das"
4. Clicks "Use in sentence"
5. ???

### Possible Behaviors

**A) Replace + re-synthesize full sentence**
```
Original: "Te're, ku'idas lä'heb?"
After:    "Te're, kui'das lä'heb?"

→ Synthesize new full sentence
→ User hears: [plays full sentence with new pronunciation]
```
- **Implication:** Stress marker must NOT "fix" the custom stress
- **Need:** Way to force specific pronunciation

**B) Replace + show + manual trigger**
```
Original: "Te're, ku'idas lä'heb?"
After:    "Te're, kui'das lä'heb?"

→ Show text updated
→ User must click "Listen" to hear it
```
- **Clearer UX:** User controls when to synthesize

**C) Hybrid audio**
```
Extract: "Tere" from original
Synthesize: "kui'das" (new variant)
Extract: "läheb" from original

→ Concatenate into new sentence audio
```
- **Pros:** Fast, no full re-synthesis
- **Cons:** May sound unnatural at boundaries
