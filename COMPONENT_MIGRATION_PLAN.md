# Frontend Migration Plan: EKI → HAK

## Цель
Заменить HAK frontend на EKI прототип, затем наполнить функциональностью и тестами.

## Подход
**Простой и чистый**: Спрятать старый frontend, запустить прототип, постепенно добавлять функционал.

---

## Фаза 1: Подготовка
- [ ] Спрятать текущий HAK frontend в `.old-frontend/`
- [ ] Скопировать EKI прототип в `src/`
- [ ] Адаптировать конфиги (vite, tsconfig) под EKI структуру

## Фаза 2: Запуск прототипа
- [ ] Исправить импорты и пути
- [ ] Запустить dev server
- [ ] **Визуальная проверка** - видим работающий красивый прототип

## Фаза 3: Backend интеграция
- [ ] Подключить HAK auth сервис
- [ ] Подключить HAK API клиенты (vabamorf, merlin, tasks)
- [ ] Проверить все API вызовы работают

## Фаза 4: Тесты
- [ ] Написать тесты для UI компонентов
- [ ] Написать тесты для страниц
- [ ] Написать тесты для интеграций
- [ ] Достичь покрытия >= 80%

## Фаза 5: Финализация
- [ ] Удалить `.old-frontend/`
- [ ] Финальная проверка
- [ ] Merge в main

---

## Структура после миграции

```
hak/packages/frontend/
├── .old-frontend/     <- спрятанный старый HAK (временно)
├── src/
│   ├── components/    <- EKI компоненты
│   ├── styles/        <- EKI стили (уже скопированы)
│   ├── services/      <- HAK сервисы (auth, API)
│   └── ...
└── ...
```

---

## Принципы

1. **Видимый прогресс** - после каждого коммита видим работающее приложение
2. **Прототип как база** - EKI визуал = правильный визуал
3. **HAK функционал** - backend интеграции из HAK
4. **Тесты в конце** - сначала работает, потом тестируем
