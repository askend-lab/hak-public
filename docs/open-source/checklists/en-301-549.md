# EN 301 549 — EU Public Sector Accessibility Checklist

> https://www.etsi.org/deliver/etsi_en/301500_301599/301549/
> **Legally mandatory** for Estonian public sector web applications.

## Chapter 9: Web (maps to WCAG 2.1 AA)
- [ ] All WCAG 2.1 Level A and AA success criteria met (see [wcag.md](./wcag.md))
- [ ] Conformance level documented in an Accessibility Statement

## Chapter 11: Software (additional requirements)
- [ ] Platform accessibility features not disrupted by the application
- [ ] User preferences for font size, colors, contrast are respected
- [ ] No interference with assistive technology (screen readers, magnifiers)

## Chapter 12: Documentation and Support
- [ ] Product documentation (README, user guides) is accessible
- [ ] Documentation available in accessible formats (HTML, not just PDF)
- [ ] Accessibility features documented for users
- [ ] Support channels accessible to people with disabilities

## Accessibility Statement (required by EU directive)
- [ ] Published on the website in an accessible format
- [ ] Contains: conformance status, non-accessible content, feedback mechanism
- [ ] Contains: date of assessment, assessment method
- [ ] Updated annually or when significant changes are made
- [ ] Link to enforcement body (Estonian Consumer Protection and Technical Regulatory Authority)

## Testing Requirements
- [ ] Automated testing with axe-core integrated into CI
- [ ] Manual testing with keyboard-only navigation
- [ ] Testing with at least one screen reader (NVDA or VoiceOver)
- [ ] Testing at 200% zoom level
- [ ] Testing at 320px viewport width
