## 1. State Management — внедрение Zustand

### Проблема
- 10+ взаимосвязанных hooks в `features/synthesis/hooks/synthesis/`
- `useSynthesis.ts` собирает их в мега-hook с 20+ методами
- Нет глобального state management

### Решение
Внедрить **Zustand** для глобального состояния synthesis.

### План
1. [ ] Создать `packages/frontend/src/stores/synthesisStore.ts`
2. [ ] Перенести состояние из `useSentenceState.ts` в store
3. [ ] Перенести audio state из `useAudioPlayer.ts` в store
4. [ ] Рефакторить `useSynthesisOrchestrator.ts` на использование store
5. [ ] Упростить `useSynthesis.ts` до thin wrapper над store
6. [ ] Удалить неиспользуемые hooks

### Критерий успеха
Код становится проще и понятнее. Если нет — не переносим.

### Оценка
- **Сложность:** High
- **Время:** 2-3 дня
- **Риск:** Medium (много тестов завязано на текущую структуру)

## 2. Реорганизация .partN тестов

### Проблема
- 409 тестовых файлов, многие разбиты на `.part2.test.ts`, `.part3.test.ts`
- Разбиение выглядит искусственным

### Причина разбиения
Лимит ~400-500 строк на файл для корректной работы AI-ассистентов.

### Решение
Разбить тесты на **логические группы** вместо `.partN`:
- `useSynthesis.mutations.test.ts`
- `useSynthesis.playback.test.ts`
- `useSynthesis.state.test.ts`

### План
1. [ ] Найти все `.partN.test.ts` файлы
2. [ ] Определить логические группы тестов в каждом
3. [ ] Переименовать файлы по функциональности
4. [ ] Обновить импорты если нужно

### Оценка
- **Сложность:** Medium
- **Время:** 1-2 дня
- **Риск:** Low

## 4. Документация Merlin TTS

### Проблема
- Merlin TTS — устаревший синтезатор (2016)
- Независимый разработчик спросит "почему не Coqui/VITS?"

### Решение
Документировать причину выбора Merlin.

### План
1. [x] Создать `docs/adr/009-merlin-tts-engine.md`
2. [x] Объяснить: используем обученную модель от EKI
3. [x] Описать ограничения и возможные альтернативы
4. [ ] Добавить ссылку в ARCHITECTURE.md

### Содержание ADR
- Merlin — единственный движок с обученной эстонской моделью от EKI
- Обучение новой модели не входит в scope проекта
- При появлении лучшей модели — можно заменить worker

### Оценка
- **Сложность:** Low
- **Время:** 0.5 дня
- **Риск:** None
