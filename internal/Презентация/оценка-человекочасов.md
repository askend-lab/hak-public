# Экспертная оценка: сколько человекочасов занял бы проект HAK

## Методика оценки

Оценка основана на:
1. Фактический объём кода и компонентов проекта
2. Индустриальные метрики производительности (COCOMO II, средние данные по отрасли)
3. Сложность компонентов (не только строки, но и интеграции, инфраструктура, тесты)

**Средняя продуктивность разработчика:** 50-100 строк production-ready кода в день (с учётом тестов, ревью, отладки, документации). Для сложных систем (AWS, ML, безопасность) — ближе к нижней границе.

## Итоговая оценка

### Оптимистичная: ~1 600 человекочасов
Предполагает опытную команду, знакомую с AWS/React/Terraform, минимум переделок.

### Реалистичная: ~2 000 человекочасов
Типичная команда middle+ уровня, нормальное количество итераций и отладки.

### Пессимистичная: ~2 500 человекочасов
Команда с меньшим опытом AWS/инфраструктуры, дополнительное время на обучение и координацию.

## Сравнение: AI vs человек

| Метрика | AI (факт) | Человек (оценка) |
|---------|-----------|------------------|
| Календарное время | 3 месяца | 4-6 месяцев (3 чел.) |
| Человекочасы | ~400-500 ч (Алекс) | ~2 000 ч (команда) |
| Стоимость разработчиков | 1 человек + AI подписки | 3 разработчика × зарплата |
| Коммиты | 2 320 | ~500-800 (типично) |
| PR | 701 | ~200-300 (типично) |
| Тестовое покрытие | 80-95% | 60-80% (типично) |

### Экономия

При средней ставке senior-разработчика в Эстонии (~40-50 €/час):

- **Стоимость человеческой команды:** 2 000 ч × 45 € = **~90 000 €**
- **Стоимость AI-подхода:** 500 ч Alex + AI подписки + AWS CI/CD ≈ **значительно меньше**

## Оговорки

1. **ML-модель Merlin** — 567K строк Python не считаются в оценку, т.к. это внешняя библиотека. Считается только интеграция.
2. **Качество первой итерации** — AI-код потребовал 2 код-ревью и множество исправлений. Человек мог бы написать лучше с первого раза, но дольше.
3. **Инфраструктура** — оценка предполагает, что разработчики знакомы с AWS. Для команды без опыта AWS время может удвоиться.
4. **Тесты** — 407 тестовых файлов и 37K+ строк тестов frontend — это много. Типичная человеческая команда написала бы меньше тестов за то же время.
